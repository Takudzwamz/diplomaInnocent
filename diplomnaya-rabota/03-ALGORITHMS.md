# 03 — Алгоритмы рекомендательной системы

## Обзор стратегий

Система реализует **4 стратегии** рекомендаций, каждая из которых может быть назначена пользователю через механизм A/B тестирования:

| # | Стратегия | Описание | Использование |
|---|-----------|----------|---------------|
| 1 | **Popular** | Глобально популярные товары | Контрольная группа, холодный старт |
| 2 | **CollaborativeFiltering** | «Пользователи, похожие на вас, купили...» | Пользователи с историей |
| 3 | **ContentBased** | Товары, похожие по содержанию (эмбеддинги) | Страницы товаров |
| 4 | **Adaptive (Гибридная)** | Взвешенная комбинация всех подходов | Основная группа |

---

## 1. Стратегия Popular (Популярные)

**Файл:** `Infrastructure/Services/AdaptiveRecommendationService.cs`

### Алгоритм:

1. Получение всех взаимодействий за последние 30 дней
2. Применение весов к каждому типу взаимодействия
3. Группировка по `ProductId`, суммирование весов
4. Сортировка по убыванию → Топ-K товаров

### Веса взаимодействий:

| Тип взаимодействия | Вес |
|--------------------|-----|
| Purchase (Покупка) | 5 |
| AddToCart (В корзину) | 3 |
| Click (Клик) | 2 |
| RecommendationClick | 2 |
| Wishlist (Избранное) | 2 |
| View (Просмотр) | 1 |

### Формула оценки товара:

$$
\text{PopularityScore}(p) = \sum_{i \in \text{interactions}(p)} w(i.\text{type})
$$

где $w(\text{type})$ — вес из таблицы выше.

## 2. Коллаборативная фильтрация (Collaborative Filtering)

**Файл:** `Infrastructure/Services/AdaptiveRecommendationService.cs`

### Алгоритм:

1. Получить множество товаров текущего пользователя: $P_u$
2. Найти пользователей с пересечением по товарам (соседей)
3. Для каждого соседа $v$: товары, которые он оценил, но текущий пользователь не видел
4. Оценка каждого кандидата = сумма взвешенных сигналов от всех соседей

### Формула:

$$
\text{CFScore}(u, p) = \sum_{v \in \text{Neighbors}(u)} \sum_{i \in \text{interactions}(v, p)} w(i.\text{type})
$$

где:
- $\text{Neighbors}(u)$ — множество пользователей, имеющих взаимодействия хотя бы с одним товаром из $P_u$
- $w(i.\text{type})$ — вес типа взаимодействия (та же таблица)
- Рассматриваются только товары $p \notin P_u$ (исключаем уже виденные)

### Ограничения:
- Максимум **50** кандидатов товаров
- Возвращается Топ-K по CFScore

---

## 3. Контентная фильтрация (Content-Based)

**Файл:** `Infrastructure/Services/AIRecommendationService.cs`

### Алгоритм:

1. Загрузить вектор эмбеддинга текущего товара: $\vec{e}_p$ (1536 измерений)
2. Загрузить все эмбеддинги товаров из базы (параллельно)
3. Вычислить косинусное сходство между $\vec{e}_p$ и каждым кандидатом $\vec{e}_q$
4. Отсечь товары с `similarity < 0.1`
5. Вернуть Топ-K по убыванию сходства

### Формула косинусного сходства:

$$
\text{CosineSimilarity}(\vec{a}, \vec{b}) = \frac{\vec{a} \cdot \vec{b}}{|\vec{a}| \cdot |\vec{b}|} = \frac{\sum_{i=1}^{1536} a_i \cdot b_i}{\sqrt{\sum_{i=1}^{1536} a_i^2} \cdot \sqrt{\sum_{i=1}^{1536} b_i^2}}
$$

### Генерация эмбеддингов:

**Файл:** `Infrastructure/Services/ProductEmbeddingService.cs`

- Модель: `text-embedding-ada-002` (Azure OpenAI)
- Размерность: 1536
- Входной текст: `"{Name} {Description} {Brand} {Type} {Price:F2}"`
- Генерация: при старте приложения для товаров без эмбеддингов
- Хранение: колонка `Embedding` в таблице `Products` (JSON-массив float[])

## 4. Гибридная адаптивная стратегия (Adaptive)

**Файл:** `Infrastructure/Services/AdaptiveRecommendationService.cs`

### Алгоритм:

1. Определить уровень «прогрева» пользователя (количество взаимодействий)
2. Получить оценки от каждой подсистемы
3. Применить формулу взвешенного комбинирования
4. Отсортировать финальные оценки → Топ-K

### Веса гибридной модели:

| Компонент | Вес | Описание |
|-----------|-----|----------|
| Collaborative Filtering | 0.40 | Оценки от похожих пользователей |
| Content-Based | 0.35 | Косинусное сходство эмбеддингов |
| Trending | 0.15 | Популярность за последние 7 дней |
| Recency | 0.10 | Свежие товары (новинки) |

### Формула финальной оценки:

$$
\text{HybridScore}(u, p) = 0.40 \cdot \text{CF}(u,p) + 0.35 \cdot \text{CB}(u,p) + 0.15 \cdot \text{Trend}(p) + 0.10 \cdot \text{Recency}(p)
$$

### Обработка холодного старта (Cold-Start):

| Количество взаимодействий | Стратегия |
|--------------------------|-----------|
| 0 | 100% Popular |
| 1–2 | 70% Popular + 30% Content-Based |
| ≥ 3 | Полная гибридная формула |

Логика:
```csharp
if (interactionCount == 0)
    return GetPopularRecommendations(count);
else if (interactionCount < 3)
    // Смешение: 70% Popular, 30% Content-Based
else
    // Полная гибридная формула (4 компонента)
```

## Нормализация оценок

Перед объединением все оценки нормализуются в диапазон [0, 1]:

$$
\text{Normalized}(s) = \frac{s - s_{\min}}{s_{\max} - s_{\min}}
$$

Это гарантирует, что каждый компонент вносит вклад пропорционально своему весу, независимо от абсолютных значений.

---

## Расчёт Trending (Трендовых)

Оценка трендовости учитывает популярность за последние 7 дней:

$$
\text{TrendScore}(p) = \sum_{i \in \text{interactions}(p, 7\text{дней})} w(i.\text{type})
$$

---

## Расчёт Recency (Новизны)

Оценка новизны основана на дате добавления товара:

$$
\text{RecencyScore}(p) = \max\left(0,\ 1 - \frac{\text{DaysSinceAdded}(p)}{30}\right)
$$

Товары старше 30 дней получают оценку 0 по этому компоненту.

## Сравнение стратегий

| Критерий | Popular | CF | Content-Based | Adaptive |
|----------|---------|-----|---------------|----------|
| Персонализация | Нет | Высокая | Средняя | Высокая |
| Холодный старт | ✅ Работает | ❌ Нет данных | ✅ Работает | ✅ Адаптируется |
| Разнообразие | Низкое | Среднее | Низкое | Высокое |
| Serendipity | Низкое | Высокое | Низкое | Среднее |
| Вычислительная сложность | O(n) | O(n·m) | O(n·d) | O(n·m + n·d) |

где $n$ — количество товаров, $m$ — количество пользователей, $d$ — размерность эмбеддинга (1536).
