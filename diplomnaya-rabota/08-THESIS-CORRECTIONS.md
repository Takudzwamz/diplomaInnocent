## Глава 3: Описание гибридного алгоритма

### Вставить/заменить описание:

---

Разработанная гибридная адаптивная стратегия рекомендаций объединяет четыре компонента в единую формулу:

$$\text{HybridScore}(u, p) = 0{,}40 \cdot CF(u,p) + 0{,}35 \cdot CB(u,p) + 0{,}15 \cdot Trend(p) + 0{,}10 \cdot Recency(p)$$

где:
- $CF(u,p)$ — оценка коллаборативной фильтрации (сумма взвешенных сигналов от похожих пользователей);
- $CB(u,p)$ — косинусное сходство эмбеддингов товаров (вектора размерностью 1536, модель text-embedding-ada-002);
- $Trend(p)$ — популярность товара за последние 7 дней;
- $Recency(p)$ — свежесть товара (линейное затухание за 30 дней).

Для решения проблемы **холодного старта** используется каскадная стратегия:
- При 0 взаимодействий — 100% Popular (глобальная популярность);
- При 1–2 взаимодействиях — смесь 70% Popular + 30% Content-Based;
- При 3+ взаимодействиях — полная гибридная формула.

## Глава 3: A/B тестирование

### Вставить:

---

Для экспериментальной оценки стратегий реализован механизм A/B тестирования с детерминированным назначением пользователей в группы. Алгоритм использует хеш-функцию:

```
hash = 17
для каждого символа c в строке "{userId}-{experimentId}":
    hash = hash × 31 + код_символа(c)
группа = |hash| mod 100 < процент_трафика ? Эксперимент : Контроль
```

Свойство детерминированности гарантирует, что повторные визиты пользователя всегда приводят к одной и той же группе без необходимости предварительного назначения.

Статистическая значимость результатов проверяется:
- **z-тестом** для пропорций (CTR, конверсия);
- **t-тестом Уэлча** для средних значений (средний чек);
- Уровень значимости $\alpha = 0{,}05$.

## Глава 4: Offline-метрики

### Вставить:

---

Offline-оценка выполняется методом **временного разделения** (temporal split):
- 80% данных по дате — обучающая выборка;
- 20% данных — тестовая выборка.

Рассчитываемые метрики:
- **Precision@K** — доля релевантных среди K рекомендаций;
- **Recall@K** — доля найденных релевантных из всех релевантных;
- **NDCG@K** — учёт позиции ($DCG = \sum \frac{rel_i}{\log_2(i+1)}$);
- **MRR** — средняя обратная позиция первого релевантного;
- **Coverage** — доля каталога, затронутая рекомендациями.
