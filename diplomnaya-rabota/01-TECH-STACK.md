## Структура проекта

```
skinet.sln
├── Core/                     # Доменный слой (сущности, интерфейсы, спецификации)
│   ├── Entities/             # Product, Order, UserInteraction, ABTestExperiment и др.
│   ├── Interfaces/           # Контракты сервисов (IAdaptiveRecommendationService и др.)
│   ├── Specifications/       # Паттерн Specification для запросов
│   └── Paging/               # Помощники пагинации
│
├── Infrastructure/           # Доступ к данным + внешние сервисы
│   ├── Data/                 # EF Core DbContext, Миграции, Сид-данные
│   ├── Services/             # Все реализации сервисов
│   │   ├── AdaptiveRecommendationService.cs    ← Гибридный движок рекомендаций
│   │   ├── AIRecommendationService.cs          ← Рекомендации на основе косинусного сходства
│   │   ├── ProductEmbeddingService.cs          ← Генерация эмбеддингов через Azure OpenAI
│   │   ├── ABTestService.cs                    ← Управление A/B тестами
│   │   ├── RecommendationMetricsService.cs     ← Online-метрики + статистические тесты
│   │   ├── OfflineMetricsService.cs            ← Оценка Precision@K, NDCG@K
│   │   ├── UserInteractionService.cs           ← Отслеживание поведения
│   │   └── RecommendationDataSeeder.cs         ← Генерация синтетических данных
│   └── Config/               # Конфигурации сущностей EF Core
│
├── StorefrontRazor/          # Веб-приложение (UI + контроллеры)
│   ├── Pages/                # Razor Pages
│   │   ├── Index.cshtml      # Главная с персонализированными рекомендациями
│   │   ├── Products/         # Каталог, страница товара с ИИ-рекомендациями
│   │   ├── Admin/            # Админ-панель
│   │   │   ├── Recommendations/  # Управление A/B тестами, offline-оценка
│   │   │   ├── AIInsights/       # Дашборд ИИ-аналитики
│   │   │   └── Settings/         # Настройки ИИ
│   │   └── Account/          # Страницы авторизации
│   ├── wwwroot/js/           # Клиентские скрипты отслеживания
│   └── Program.cs            # Регистрация DI, middleware, запуск
│
├── API/                      # Проект REST API (альтернативный фронтенд)
└── docker-compose.yml        # SQL Server 2022 + Redis
```

## Что заменить в диссертации

### Глава 3, Раздел 3.1.1 (Серверная часть)
**Убрать:**
> "Python / Node.js", "REST API"

**Заменить на:**
> Серверная часть реализована на платформе ASP.NET Core 10.0 с использованием языка программирования C# 13. Архитектура построена по принципу Clean Architecture с выделением трёх слоёв: Core (доменная модель), Infrastructure (доступ к данным и внешние сервисы) и StorefrontRazor (веб-приложение). Для работы с базой данных используется Entity Framework Core 10.0. Аутентификация реализована через ASP.NET Core Identity. Оповещения в реальном времени обеспечиваются через SignalR.

### Глава 3, Раздел 3.1.2 (Клиентская часть)
**Убрать:**
> "React / Vue.js", "одностраничного приложения (SPA)"

**Заменить на:**
> Клиентская часть реализована с использованием серверного рендеринга (SSR) на основе ASP.NET Razor Pages. Пользовательский интерфейс построен на фреймворке Bootstrap 5, обеспечивающем адаптивность под различные устройства. Динамическое взаимодействие, включая отслеживание кликов и показов рекомендаций, реализовано на JavaScript с использованием AJAX-запросов и IntersectionObserver API.

### Глава 3, Раздел 3.1.3 (База данных)
**Убрать:**
> "PostgreSQL", "NoSQL-хранилище (например, Redis)"

**Заменить на:**
> Для хранения данных используется реляционная СУБД Microsoft SQL Server 2022, развёрнутая в Docker-контейнере. В качестве распределённого кэша применяется Redis, обеспечивающий быстрый доступ к результатам рекомендаций и сессиям пользователей. Взаимодействие с базой данных осуществляется через Entity Framework Core с поддержкой миграций.
