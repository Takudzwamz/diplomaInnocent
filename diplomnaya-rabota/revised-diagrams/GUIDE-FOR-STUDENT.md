# Руководство: Упрощённые диаграммы для диплома

## Зачем мы переделали диаграммы?

Комиссия сказала, что старые диаграммы непонятны. Проблема была в том, что они содержали слишком много технических деталей (сигнатуры методов, все поля всех классов, внутренние интерфейсы). Комиссии это не нужно — им нужно понять ИДЕЮ.

Новые диаграммы:
- Простые (максимум 5-7 блоков)
- Всё на русском
- Без кода и сигнатур методов
- Каждая отвечает на ОДИН вопрос

## Список диаграмм и куда их ставить в дипломе

### Диаграмма 1: Общая архитектура системы
**Файл:** `01_архитектура_общая.png`

**Что показывает:** Три главных слоя приложения: Пользователь → Веб-приложение → База данных + Внешние сервисы

**Куда поставить в дипломе:** Глава 2, раздел «Архитектура системы» (сразу после описания технологий)

**Как объяснить комиссии:**
> «Система построена на трёхуровневой архитектуре. Пользователь работает через браузер. Razor Pages формирует HTML на сервере. Данные хранятся в SQL Server, а сессии корзины — в Redis. Для генерации эмбеддингов используется Azure OpenAI.»

**Файл:** `02_бд_рекомендации.png`

**Что показывает:** 5 таблиц, которые нужны для рекомендательной системы, и связи между ними.

**Куда поставить в дипломе:** Глава 2, раздел «Проектирование базы данных»

**Как объяснить комиссии:**

RecommendationEvents фиксирует, какие рекомендации показывались и по какой стратегии. ABTestExperiments и ABTestAssignments нужны для проведения A/B тестирования.»

### Диаграмма 3: Как работает рекомендация (блок-схема)
**Файл:** `03_алгоритм_рекомендации.png`

**Что показывает:** Пошаговый процесс: пользователь открывает страницу → система проверяет, есть ли у него история → выбирает алгоритм → выдаёт товары.

**Куда поставить в дипломе:** Глава 3, раздел «Алгоритм генерации рекомендаций»

**Как объяснить комиссии:**
> «Когда пользователь открывает страницу, система проверяет, есть ли у него история взаимодействий. Если нет (холодный старт) — показываем популярные товары. Если есть — гибридный алгоритм комбинирует коллаборативную фильтрацию, контентный анализ, тренды и категории пользователя. Каждый компонент даёт балл, баллы суммируются с весами, и выдаются товары с наивысшим баллом.»

### Диаграмма 4: Формула гибридного алгоритма
**Файл:** `04_формула_гибрид.png`

**Что показывает:** Визуальная формула: Score = 0.4×CF + 0.35×CB + 0.15×Trending + 0.1×Recency

**Куда поставить в дипломе:** Глава 3, раздел «Гибридный адаптивный алгоритм» (сразу после блок-схемы)

**Как объяснить комиссии:**
> «Гибридная модель объединяет четыре компонента с фиксированными весами. Коллаборативная фильтрация имеет наибольший вес (40%), потому что "похожие пользователи" — самый надёжный сигнал. Контентный анализ (35%) находит похожие товары по описанию через ИИ-вектора. Тренды (15%) и категории пользователя (10%) дополняют результат.»

### Диаграмма 5: Процесс A/B тестирования
**Файл:** `05_ab_тест.png`

**Что показывает:** Как пользователи делятся на две группы и получают разные алгоритмы рекомендаций.

**Куда поставить в дипломе:** Глава 4, раздел «Методология A/B тестирования»

**Как объяснить комиссии:**
> «Для проверки гипотезы мы используем A/B тестирование. Все пользователи случайно делятся на две группы по 50%. Контрольная группа получает рекомендации по алгоритму Popular (просто популярные товары). Экспериментальная группа получает рекомендации по адаптивному гибридному алгоритму. Затем мы сравниваем метрики: CTR и конверсию.»

## Как рассказывать комиссии

**Правило 1:** Не читай с листа. Покажи картинку и скажи 2-3 предложения своими словами.

**Правило 2:** Если спросят «как это работает?» — отвечай по блок-схеме (диаграмма 3), показывая пальцем на каждый шаг.

**Правило 3:** Если спросят «зачем A/B тест?» — скажи: «Чтобы научно доказать, что мой алгоритм лучше простого. Без A/B теста это просто слова, с A/B тестом — это эксперимент.»

**Правило 4:** Если спросят про формулу — покажи диаграмму 4 и скажи: «У меня 4 компонента, каждый вносит свой вклад. Вот их веса. Итоговый балл — это сумма.»

**Правило 5:** Если спросят про базу — покажи ER-диаграмму (диаграмма 2) и скажи: «Вот 5 таблиц для рекомендаций. UserInteractions записывает всё, что делает пользователь. На этих данных работают все алгоритмы.»
